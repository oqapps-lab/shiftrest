## MVP (Must Have) — Запуск

### F1: Ввод расписания смен

| Поле | Описание |
|------|----------|
| **Название** | Shift Schedule Input |
| **User Story** | Как медсестра с 3x12 ротацией, я хочу ввести своё расписание смен, чтобы приложение знало, когда я работаю и могло спланировать мой сон |
| **Acceptance Criteria** | - Ручной ввод: выбор дат + время начала/конца смены |
|  | - Шаблоны ротаций: 3x12 (день/ночь), 24/48, 48/96, continental (2D/2N/4off) |
|  | - Повтор расписания на N недель вперёд |
|  | - Редактирование отдельных смен (переработки, подмены) |
|  | - Визуальный календарь с цветовой кодировкой (день/ночь/выходной) |
| **Экран** | Schedule (главный экран) |
| **Приоритет** | P0 — без расписания нет продукта |
| **Сложность** | L |

### F2: Генерация плана сна

| Поле | Описание |
|------|----------|
| **Название** | Sleep Plan Generator |
| **User Story** | Как сменный работник, я хочу получить план «когда спать» на основе моего расписания, чтобы оптимизировать свой сон в рамках реальных ограничений |
| **Acceptance Criteria** | - Optimal sleep window для каждого дня (начало, конец, длительность) |
|  | - Учёт: время начала/конца смены, дорога домой (~30 мин default, настраиваемо) |
|  | - Рекомендация pre-shift nap (если применимо) |
|  | - Визуальная timeline: смена → дорога → сон → подъём → смена |
|  | - Научная основа: Two-process model, chronotype adjustment |
|  | - План на весь блок смен + выходные |
| **Экран** | Sleep Plan (второй главный экран) |
| **Приоритет** | P0 — core value proposition |
| **Сложность** | XL |

### F3: Transition Plan

| Поле | Описание |
|------|----------|
| **Название** | Night-Day Transition Planner |
| **User Story** | Как медсестра, после блока из 3 ночных смен я хочу получить пошаговый план перехода на дневной режим, чтобы не быть «зомби» на первый выходной |
| **Acceptance Criteria** | - Пошаговый план на 2-3 дня: когда спать, когда бодрствовать |
|  | - Два типа: ночь→день и день→ночь |
|  | - Учёт: хронотип, семейные обязательства (дети из школы в 15:00) |
|  | - Рекомендации по свету: когда носить тёмные очки, когда искать свет |
|  | - Рекомендация по napping: когда и сколько вздремнуть |
|  | - Таймлайн с этапами перехода |
| **Экран** | Transition (подраздел Sleep Plan) |
| **Приоритет** | P0 — killer feature, главный дифференциатор ([RESEARCH-BRIEF.md](../01-research/RESEARCH-BRIEF.md), Insight #3) |
| **Сложность** | XL |

### F4: Caffeine Cutoff Timer

| Поле | Описание |
|------|----------|
| **Название** | Caffeine Cutoff Timer |
| **User Story** | Как пожарный, я хочу знать, когда выпить последний кофе во время смены, чтобы он не помешал мне заснуть дома |
| **Acceptance Criteria** | - Автоматический расчёт cutoff на основе: конец смены + дорога + время засыпания |
|  | - Правило: ≥6 часов до планируемого сна (настраиваемо: 5-8ч) |
|  | - Push-уведомление: «Последний кофе — до [время]» |
|  | - Учёт индивидуальной чувствительности (normal / slow metabolizer) |
|  | - Визуальный индикатор: зелёный/жёлтый/красный |
| **Экран** | Sleep Plan (карточка) + уведомление |
| **Приоритет** | P1 — высокий спрос, простая реализация |
| **Сложность** | S |

### F5: Melatonin Timing

| Поле | Описание |
|------|----------|
| **Название** | Melatonin Timing Advisor |
| **User Story** | Как медсестра, я хочу знать точное время для приёма мелатонина перед дневным сном, чтобы он был максимально эффективен |
| **Acceptance Criteria** | - Расчёт оптимального времени приёма на основе: target sleep time + PRC |
|  | - Phase advance (раннее засыпание): за 10-11ч до mid-sleep |
|  | - Phase delay (позднее засыпание): утром после пробуждения |
|  | - Push-уведомление: «Примите мелатонин сейчас» |
|  | - Рекомендация дозы: 0.5-3 мг (информация, не назначение) |
|  | - Disclaimer: «Не является медицинским назначением» |
| **Экран** | Sleep Plan (карточка) + уведомление |
| **Приоритет** | P1 — высокий спрос, требует PRC-алгоритм |
| **Сложность** | M |

### F6: Онбординг

| Поле | Описание |
|------|----------|
| **Название** | Personalization Onboarding |
| **User Story** | Как новый пользователь, я хочу за 2 минуты настроить приложение под себя, чтобы сразу увидеть полезный план |
| **Acceptance Criteria** | - Шаг 1: Выбор профессии (медсестра / пожарный / заводской / другое) |
|  | - Шаг 2: Выбор шаблона ротации или ручной ввод |
|  | - Шаг 3: Хронотип (3-5 вопросов из MEQ) |
|  | - Шаг 4: Кофеин (сколько чашек/день, чувствительность) |
|  | - Шаг 5: Мелатонин (принимаете ли) |
|  | - Шаг 6: Семья (дети, обязательства с фиксированным временем) |
|  | - Результат: первый план сна отображается сразу после онбординга |
|  | - Максимум 2 минуты, возможность пропустить шаги |
| **Экран** | Onboarding flow (4-6 экранов) |
| **Приоритет** | P0 — без онбординга нет персонализации |
| **Сложность** | M |

### F7: Push-уведомления

| Поле | Описание |
|------|----------|
| **Название** | Smart Notifications |
| **User Story** | Как сменный работник, я хочу получать напоминания в нужное время, чтобы не забывать про мелатонин, кофеиновый cutoff и время сна |
| **Acceptance Criteria** | - «Пора готовиться ко сну» — за 30 мин до target sleep time |
|  | - «Последний кофе — сейчас» — в caffeine cutoff time |
|  | - «Примите мелатонин» — в melatonin timing |
|  | - «Наденьте тёмные очки» — перед выходом после ночной смены |
|  | - Все уведомления привязаны к расписанию, не к фиксированному времени |
|  | - Настраиваемость: какие получать, за сколько минут |
| **Экран** | Настройки + системные уведомления |
| **Приоритет** | P1 — critical для retention и real-world value |
| **Сложность** | M |

### F8: AI-персонализация

| Поле | Описание |
|------|----------|
| **Название** | AI Sleep Coach |
| **User Story** | Как пользователь, я хочу получать рекомендации, адаптированные к моей конкретной ситуации, а не общие правила |
| **Acceptance Criteria** | - OpenAI API генерирует план с учётом: расписание + хронотип + семья + привычки |
|  | - Structured prompts — план всегда в рамках AASM/CDC guidelines |
|  | - Validation layer: все рекомендации проходят rule-based проверку |
|  | - Fallback: если AI недоступен → rule-based алгоритм |
|  | - «Объясни почему»: каждая рекомендация с кратким обоснованием |
|  | - Стоимость: $0.01-0.05/запрос |
| **Экран** | Интегрировано в Sleep Plan |
| **Приоритет** | P1 — конкурентное преимущество, но rule-based fallback работает |
| **Сложность** | XL |

|------|----------|
| **Название** | Shift-Appropriate Sleep Score |
| **User Story** | Как сменный работник, я хочу видеть оценку качества сна, которая учитывает мой график, а не штрафует за «неправильное» время |

## Won't Have (сейчас)

| Фича | Почему НЕ делаем |
|------|-----------------|
| **Sleep tracking (собственный)** | Мы планировщик, не трекер. Tracking = другой продукт, другой рынок. Для отслеживания используем HealthKit/Health Connect |
| **Диагностика SWSD** | Юридический риск (FDA medical device). ShiftRest — wellness, не медицина ([DOMAIN-RESEARCH.md](../01-research/DOMAIN-RESEARCH.md), раздел 3.1) |
| **Prescription medication advice** | Не можем рекомендовать modafinil, зопиклон и т.д. — только OTC мелатонин (информация, не назначение) |
| **B2B Dashboard** | Year 2 приоритет. Сначала доказываем product-market fit на consumer ([RESEARCH-BRIEF.md](../01-research/RESEARCH-BRIEF.md), рекомендация #6) |
| **Social features** | Не habit tracker. Social accountability не релевантна для sleep planning |
| **Meditation / Sound Machine** | Уже перенасыщенный рынок (Calm, Headspace). Не наш дифференциатор |
| **Wearable requirement** | Приложение должно работать без wearable. Wearable — опциональный бонус ([RESEARCH-BRIEF.md](../01-research/RESEARCH-BRIEF.md), рекомендация #6) |
| **Multi-language** | MVP — English only. Локализация — v2.0+, после подтверждения PMF |

## MVP Scope Summary

### Экраны MVP

| # | Экран | Фичи | Приоритет |
|---|-------|-------|-----------|
| 1 | Onboarding (4-6 шагов) | F6 | P0 |
| 2 | Schedule (календарь смен) | F1 | P0 |
| 3 | Sleep Plan (timeline) | F2, F3, F4, F5, F8 | P0/P1 |
| 4 | Transition Plan (детальный) | F3 | P0 |
| 5 | Settings | F7, профиль, подписка | P1 |

**Итого: 5 основных экранов + онбординг**

### Сводка MVP

| Параметр | Значение |
|----------|----------|
| **Фич (Must Have)** | 8 (F1-F8) |
| **Экранов** | 5 + онбординг (4-6 шагов) = ~10-11 уникальных экранов |
| **P0 (блокеры запуска)** | 4 фичи: Schedule Input, Sleep Plan, Transition Plan, Onboarding |
| **P1 (нужно к запуску)** | 4 фичи: Caffeine Timer, Melatonin Timing, Notifications, AI Coach |
| **Оценка сложности** | 2x XL + 1x L + 3x M + 1x S = ~16-20 недель (один разработчик) |
| **Целевой срок** | 4-6 месяцев от старта дизайна до App Store |

### Dependency Map

```
F6 (Onboarding) → F1 (Schedule) → F2 (Sleep Plan) → F3 (Transition Plan)
                                        ↓
                                   F4 (Caffeine)
                                   F5 (Melatonin)
                                   F8 (AI Coach)
                                        ↓
                                   F7 (Notifications)
```

**Критический путь:** Onboarding → Schedule Input → Sleep Plan Algorithm → Transition Planning
